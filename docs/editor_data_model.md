# 게임 데이터 편집기 구조

`editor/server.py`는 한국어 최신 빌드나 일본어 원본을 읽고, 검증된 ROM
필드만 수정한 별도 ROM을 만듭니다. 출력은
`roms/builds/Langrisser II (Korean Editor Edit).md`이며 기준 ROM을
덮어쓰지 않습니다.

## 시나리오 배치

- 포인터 표: `0x18005E`
- 각 시나리오 헤더 `+0x0C`: 고정 배치 목록 포인터
- 레코드 크기: 36바이트
- 편집 필드: LV `+0x0E`, AT/DF `+0x12/+0x13`, 클래스 `+0x1B`,
  용병 6칸 `+0x1E..+0x23`
- 읽기 전용: 이름 ID, 좌표, 진영, 이벤트 플래그

좌표와 진영은 이벤트가 실행 중에 다시 쓸 수 있으므로 아직 편집하지
않습니다. 클래스 ID만 바꿔도 런타임 마법·소환 권한이 자동으로 붙지는
않습니다.

## 아이템

- 효과 표: `0x060530`
- 아이템당 8바이트: `(효과 종류, 값)` 4쌍
- 가격 표: `0x0A1D32`
- 가격 단위: ROM 값 1당 게임 표시 10P

일반 효과 종류는 AT, DF, MV, 지휘범위, A/D보정, 마법 사거리·대미지,
마법저항, 소환 ID입니다. 경험치 2배, 공격 사거리, 룬스톤 클래스 초기화,
MP 2배 같은 일부 아이템 동작은 개별 코드에 하드코딩되어 있어 현재
편집기에서는 읽기 전용 참고 정보로만 표시합니다.

아이콘은 `captures/run/97e0_item_id01.png`부터 `id37.png`까지의 검증된
320x240 상점 캡처에서 선택 행의 16x16 영역을 잘라 생성합니다.
`tools/build_item_icon_assets.py`가 같은 좌표 계산과 37개 완전성을
검증합니다. 이 PNG는 에디터 표시 자산이며 ROM 그래픽을 수정하지 않습니다.

## 클래스 체인지

- 지휘관별 포인터 표: `0x08253A`
- 지휘관: 엘윈부터 제시카까지 10명
- 지휘관당 전이: 일반 9개와 최종 1개
- 일반 전이: 현재 클래스 1개와 후보 3개
- 최종 전이: 현재 클래스 1개와 후보 1개, 뒤에 `FFFF`

ROM의 10개 전이 레코드는 10단계가 아니다. 화면은 다음 소유 관계를
실제 최대 5단계 트리로 펼칩니다.

1. 레코드 0: 1단계 시작 클래스에서 2단계 후보 3개
2. 레코드 1..3: 각 2단계 클래스에서 3단계 후보
3. 레코드 4..8: 각 3단계 클래스에서 4단계 후보
4. 레코드 9: 지정된 4단계 클래스에서 숨은 5단계 후보

현재 클래스가 한 지휘관 경로 안에서 중복되거나 클래스 ID 범위를 벗어나면
빌드를 거부합니다.

### 전직 용병 해금

- 클래스 레코드 표: `0x05EDDC`
- 레코드 크기: `0x1C`
- 새 용병 비트 인덱스: `+0x1A`, `+0x1B`
- `FF`: 해금 없음
- 저장값 `00..0F`: 실제 용병 클래스 ID `62..71`

`0x014C68..0x014CAC`의 전직 적용 코드는 두 비트를 지휘관의 기존 고용
비트마스크에 추가합니다. 따라서 에디터의 두 칸은 그 클래스가 전직 시
**새로** 해금하는 용병이며, 전체 고용 목록을 덮어쓰지 않습니다.

### 클래스와 용병 그림

- 일반 클래스 스프라이트 ID 표: `0x05DDE6`, 157개 워드
- 지휘관별 스프라이트 포인터 표: `0x05DB80`, 10개 포인터
- 16x16 스프라이트 그래픽 시작: `0x052980`
- 스프라이트당 128바이트, 8x8 타일 4개

`tools/build_class_sprite_assets.py`는 일본판 ROM에서 일반 클래스 157종의
4개 원본 팔레트와 10명 지휘관의 클래스별 전용 그림을 추출합니다.
시나리오와 클래스 트리는 지휘관 전용 그림을 우선하고, 해당 클래스
오버라이드가 없으면 같은 ID의 일반 그림으로 대체합니다. 이전
`class-previews`는 후보 화면의 용병 편성 영역을 자른 것이어서 캐릭터
클래스 그림으로 사용하지 않습니다. 외부 클래스 체인지 표와 이미지
모음은 이름·트리의 육안 대조에만 사용하고 ID 소유권은 ROM을 기준으로
결정합니다.

## 통합 빌드

`POST /api/build`는 한 번의 요청에서 다음 데이터를 순서대로 적용합니다.

1. 불러온 모든 시나리오 수정
2. 37개 아이템 가격과 효과
3. 10명 지휘관의 클래스 체인지 경로
4. 157개 클래스의 전직 용병 해금 두 칸
5. MD 헤더 체크섬 재계산

편집하지 않은 전체 데이터를 다시 전달한 회귀 검증에서는 결과 ROM이 기준
한국어 ROM과 바이트 단위로 동일했습니다. 현재 기준 체크섬은 `FDC9`,
SHA-256은
`7ad7d2199378380a6abb7bac94825799e57d7bf3f3122ded45225c7270a2b19c`
입니다.
