# 엘윈 PixelLab 네이티브 16×16 실험

이 실험은 확대된 AI 원화를 줄이지 않는다. ROM의 실제 `16×16` 로드
스프라이트를 PixelLab 인페인트 입력으로 그대로 사용한다.

- `04-source-16.png`: 편집 입력인 ROM 원본
- `04-mask-16.png`: 검정은 생성 전부터 고정할 얼굴·머리와 외부 투명
  배경, 흰색은 원본 실루엣 안의 몸·갑옷·망토·무기를 다시 그릴 영역
- `04-*-preview.png`: 마스크 확인만을 위한 16배 최근접 확대본
- `04-request.json`: 캔버스·잠금 영역·프롬프트 요청 명세

`python tools/pixellab_elwin_inpaint.py`는 입력만 다시 만든다. API 요청은
`PIXELLAB_API_TOKEN`이 설정되거나 `/home/hong/.config/pixellab/token`에
토큰이 저장된 상태에서 `--run`을 명시해야만 전송된다. 토큰 파일은
프로젝트 밖에 두며 파일 권한을 `600`으로 제한한다.
기본 실행은 로드 후보를 지정한 시드로 요청하고, 검은 마스크의
원본 픽셀이 하나라도 달라진 결과를 자동 탈락시킨다.

후보가 생성되더라도 자동으로 에디터에 적용하지 않는다. 16배 확대본과
실제 16×16 표시를 함께 보고 얼굴-목-어깨 연결과 클래스 식별성을 사람이
확인한 후보만 적용한다.

초기 실패는 아래쪽 손 피부색 덩어리를 얼굴로 오인한 마스크, 전체 배경을
편집 가능하게 둔 요청, 배경 제거 옵션이 겹친 결과였다. 교정된 마스크는
실제 위쪽 얼굴 `(10..12, 1..2)`과 갈색 머리를 포함하는
`(4,0)..(13,4)`, 그리고 원본 바깥의 투명 배경을 모두 잠근다.

시드 `1337`은 PixelLab이 네이티브 16×16에서 직접 만들고 몸 안쪽
30픽셀을 재도색했지만, 원본 실루엣 전체를 잠근 탓에 실제 크기에서 클래스
차이가 보이지 않았다. 이 후보는 에디터와 실험 폴더에서 제거하고 ROM
원본을 복원했다. 다음 실험은 얼굴·갈색 머리만 정체성 기준으로 유지하고
몸 아래의 로드 갑옷·망토·방패 실루엣은 원본과 다르게 생성해야 한다.
